# 포트폴리오 프로젝트 체크리스트

이 문서는 `포트폴리오_가이드.md`를 바탕으로 LangChain Quiz Chatbot 프로젝트 진행 시 준수해야 할 핵심 지침을 정리한 것입니다.

## 1. 프로젝트 기본 설정
- [ ] **한 줄 컨셉**: 프로젝트의 핵심 가치를 담은 문구 정의 (예: "PDF 문서를 활용한 지능형 퀴즈 자동 생성 및 학습 보조 시스템")
- [ ] **기술 스택**: LangChain, LangGraph, Gemini 1.5 Flash, FAISS, Streamlit, uv 등 사용 기술 명시

## 2. 프로젝트 핵심 구성 (포트폴리오 반영)
- [ ] **문제 정의**: 기존 학습 방식(수동 문제 풀이 등)의 문제점 및 챗봇을 통한 해결 방안 도출
- [ ] **설계 구조**: LangGraph를 이용한 상태 관리 및 RAG 시스템 아키텍처 정리
- [ ] **핵심 코드 설명**:
    - [ ] `question_generator()`: 문서 기반 퀴즈 생성 알고리즘 설명
    - [ ] `search_pdf_documents`: 벡터 스토어 검색 및 답변 생성 최적화 로직
- [ ] **성능 개선 시도**: 프롬프트 엔지니어링, 검색 품질(Top-k 등) 변경에 따른 실험 데이터 기록
- [ ] **한계 및 개선 방안**: 현재 구현의 한계점 인지 및 향후 고도화(예: 멀티모달 지원 등) 계획

## 3. 개발 및 기록 원칙
- [ ] **과정 중심 기록**: 결과물보다 문제 해결 과정과 '왜 이렇게 설계했는지'에 대한 기술적 근거 기록
- [ ] **개선 시도 강조**: 단순 기능 구현보다 성능을 높이기 위해 시도했던 실험들(예: Chunking 전략 변경)을 부각

## 4. 심화 탐구 항목 (관련 보고서)
- [ ] **비교 실험**: LLM 모델 간(Gemini Pro vs Flash) 퀴즈 생성 품질 비교 또는 응답 속도 분석
- [ ] **데이터 분석**: 문서의 복잡도에 따른 검색 정확도 변화 실험

---
> [!NOTE]
> 이 체크리스트는 프로젝트가 진행됨에 따라 업데이트하며, 포트폴리오 작성 시 각 항목이 충실히 반영되었는지 확인하는 용도로 사용합니다.
